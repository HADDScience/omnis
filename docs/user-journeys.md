# Omnis 핵심 사용자 여정 5종 (v4 §4)

> Phase 1 와이어 보존. Phase 2 구현 시 본 문서를 acceptance test 기준으로 사용.
> 출처: `docs/omnis-ux-overhaul-plan-v4.md` §4
> 검증 도구: Playwright `mobile` 프로젝트 (iPhone 13, 375x812)

각 여정에 (a) 핵심 화면 sequence (b) friction point (c) shadcn 매핑 (d) 시간 목표.

## J1. 관리자: 채팅에서 5초 안에 업무 등록

**시간 목표**: ≤ 5초 (관리자 α_instruction)

```
[채팅 dock]                [/업무 모달 열림 + AI 응답 도착]              [저장 후]
┌───────────────────┐      ┌─────────────────────────────────┐         ┌─────────────────┐
│ 김아리: ...       │      │ ✕ 업무 등록 (AI가 채워둠)        │         │ 채팅에 카드 표시 │
│ 정우창: ...       │      │ ─────────────────────────────── │         │ 🤖 업무 #abc 생성│
│ /업무 ▼          │ ────►│ 업무명: [상표권 출원 검토   ]    │ ──저장─►│ "[김아리] 정우창 │
│ ─────────────────│      │ 담당자: [정우창 ▼]              │         │  3/30 까지"      │
│ [메시지 입력...]  │      │ 우선순위: ★★☆                  │         │ [열기]           │
└───────────────────┘      │ 마감: [2026-03-30 📅]            │         └─────────────────┘
                           │ 프로젝트: [브랜드보호 ▼ → 노바스] │
                           │ 체크리스트: ☐ 선행조사            │
                           │            ☐ 명세서 작성          │
                           │ 배경: ──────                     │
                           │  [저장] [취소]                   │
                           └─────────────────────────────────┘
```

| 단계 | 시간 | Friction Point | 해결 |
|-----|------|----------------|------|
| 채팅에서 `/업무` 입력 | < 1s | 슬래시 자동완성이 모달 위 cmdk와 z 충돌 | z 토큰 (popover < dialog) |
| AI 응답 대기 (Skeleton) | 1~3s | **로딩 중 사용자 입력 → 응답 도착 시 덮어씀** | rhf `dirtyFields`로 사용자 입력 보존 |
| 속성 검수 + 수정 | 2s | **모바일에서 Combobox + DatePicker가 키보드 가림** | viewport `< 768px`은 Sheet로 풀스크린 |
| [저장] 클릭 | < 1s | optimistic update 실패 시 롤백 안 보임 | sonner toast + 인라인 inverse |

**shadcn 매핑**: `Dialog`, `Form` (rhf+zod), `Combobox` (cmdk+Popover), `Calendar`+`Popover`, `Skeleton`, `Toaster`(sonner).

## J2. 실무자: 알림 받고 업무 수행 → 체크리스트 완료

```
[notification bell]    [업무 상세 진입]                          [체크 완료 후]
┌──────────┐           ┌──────────────────────────────────────┐  ┌──────────┐
│ 🔔 새 업무│           │ 상표권 출원 검토         ★★☆  진행중│  │ 자동 완료│
│ #abc     │   클릭    │ ─────────────────────────────────── │  │ 🤖 #abc 완료│
│ 김아리   ├──────────►│ 담당: 정우창 · 마감: 3/30 · 노바스   │  │           │
│ → 정우창 │           │ ─────────────────────────────────── │  └──────────┘
└──────────┘           │ 배경: ...                            │
                       │ ☑ 선행조사  ┌─────────────┐ │
                       │ ☑ 명세서   │ 우측 Activity│ │
                       │ ☑ 출원     │ Rail         │ │
                       │ ─────────  │ ─ 스레드     │ │
                       │ 첨부 1개   │ 📎 김아리:   │ │
                       └────────────┴─────────────┘ │
```

| 단계 | Friction Point | 해결 |
|-----|---------------|------|
| 알림 → 업무 진입 | **딥링크 부재 → dashboard로 이동** | `/tasks/[id]?from=notification` 라우트 |
| 본문 가독 | **가운데 빔** | viewport 책임 표 (CLAUDE.md) |
| 체크리스트 키보드 조작 | Tab 순서 ≠ 시각 순서 | role="list" + 명시적 tabIndex |
| 마지막 체크 = 자동 DONE | **자동 완료 시 사용자 의도 미반영 시 롤백 어려움** | 토스트 + Undo 5초 |

**shadcn 매핑**: `Card`, `Checkbox`, `Tabs` (모바일), `Collapsible` (Activity Rail), `ScrollArea`, `HoverCard`.

## J3. 업무별 스레드 답장

```
[ChatDock]                          [업무 상세 진입 → 사이드바 Composer]
┌──────────────────────┐            ┌──────────────────────────┐
│ 전체  Omnis AI       │            │ 우측 Activity Rail        │
│ ──────────────────── │            │ 스레드  히스토리          │
│ #abc 정우창: 초안    │  업무 핀   │ ──────────────────────── │
│ #def 박소정: 리뷰    ├──클릭─────►│ 🟦 김아리: ...           │
│ ──────────────────── │            │ 🟩 정우창: ...           │
│ [입력...]            │            │ ─────────────────────── │
└──────────────────────┘            │ [@김아리 답장...    ⏎]   │
                                    └──────────────────────────┘
```

| Friction Point | 해결 |
|---------------|------|
| **dock에서 특정 업무만 보기 어려움** | dock에 `#업무명` 필터 chip + URL `?taskId=` |
| **상세 사이드바에 입력창 없음** (현재 제일 큰 문제) | MessageComposer 내장. `Cmd+Enter` 전송 |
| 모바일 키보드 가림 | viewport `< 768px`은 Bottom Sheet (Vaul) + safe-area-inset |

**shadcn 매핑**: `Tabs`, `Input` (filter), `Textarea` + `Button` (Composer), `Vaul Drawer` (모바일).

## J4. Omnis RAG 질의

```
[ChatDock → "Omnis AI" 탭]            [질의 후]
┌─────────────────────────┐          ┌──────────────────────────┐
│ 전체  ▶ Omnis AI ◀     │          │ 🟪 user: HADD 매출 알려줘 │
│ ─────────────────────── │          │ ─────────────────────── │
│ 제품·인증·매출·인력에   │          │ 🟦 Omnis: 2026 Q3 BEP    │
│ 대해 자유롭게 물어보세요  │          │  도달 후 ...             │
│ ─────────────────────── │          │  📎 출처:                │
│ [HADD 매출 알려줘    ⏎]│          │  ┌──────────────────┐    │
└─────────────────────────┘          │  │ 📊 재무현황 카드  │    │
                                     │  │  (Hover 미리보기) │    │
                                     │  └──────────────────┘    │
                                     └──────────────────────────┘
```

| Friction Point | 해결 |
|---------------|------|
| **citation 없으면 LLM 환각 의심** | 답변에 `📎 [출처: 카드명]` HoverCard 미리보기 필수 |
| 응답 시간 (>1s) | Skeleton + streaming token 출력 |
| 권한 (privacy) | omnis 카드 접근 권한 = 사용자별 분리 (현재 모두 공유 — 비전상 OK) |

**shadcn 매핑**: `Tabs`, `HoverCard` (citation), `Skeleton`, `Badge` (출처), `ScrollArea`.

## J5. Omnis 카테고리 직접 진입

```
[/omnis (첫 화면)]                        [/omnis/기업정보]
┌────────────────────────────┐           ┌────────────────────────────┐
│ Omnis (Cmd+K로 검색)       │           │ Omnis > 기업정보          │
│ ─────────────────────────  │           │ ─────────────────────────  │
│ ┌──────┐ ┌──────┐ ┌──────┐ │  카드 클릭 │ ┌──────┐ ┌──────┐ ┌──────┐│
│ │기업  │ │인력  │ │지식  ├─┼──────────►│ │ 회사 │ │ 재무 │ │ 인증 ││
│ │정보  │ │현황  │ │재산권│ │           │ │ 개요 │ │ 현황 │ │ 시설 ││
│ └──────┘ └──────┘ └──────┘ │           │ └──────┘ └──────┘ └──────┘│
│                            │           │                            │
│ 최근 카드:                  │           │ ↑ breadcrumb로 돌아가기    │
│ • 회사 개요 (3일 전)        │           └────────────────────────────┘
└────────────────────────────┘
```

| Friction Point (현재 문제) | 해결 |
|---------------------------|------|
| **상단에 검색 input 노출** → 검색 페이지로 인지 | 검색 input 제거, Cmd+K 키 안내 chip |
| 카테고리 진입 후 URL 변화 없음 → 새로고침 시 첫 화면 | 라우트 분리 `/omnis/[categorySlug]` |
| 카테고리 색상 8종 디자인 | oklch 토큰 (--cat-1 ~ --cat-8) |

**shadcn 매핑**: `Card` (카테고리 그리드), `Breadcrumb`, `Command` (Cmd+K), `Badge`.

## Acceptance Test (Playwright `mobile` 프로젝트)

각 여정의 모바일(375x812) 통과 기준:

| 테스트 ID | 시나리오 | 통과 조건 |
|----------|---------|----------|
| MA1 | J1 5초 등록 | native prompt 0회 + 모달 viewport 100% + 모든 입력 인라인 |
| MA2 | J2 알림 → 진입 | 딥링크 동작 + 가운데 빔 0 + Composer keyboard 활성 시 가려지지 않음 |
| MA3 | J3 스레드 입력 | 사이드바 Composer 동작 + Cmd+Enter 전송 + filter chip 동작 |
| MA4 | J4 RAG 질의 | citation 표시 + Skeleton 렌더 |
| MA5 | J5 카테고리 진입 | 검색 input 미노출 + URL 변경 + Cmd+K 동작 |

> Phase 2 종료 시점에 통과해야 라이브 머지 가능. Phase 1에서는 와이어 보존만 수행.
